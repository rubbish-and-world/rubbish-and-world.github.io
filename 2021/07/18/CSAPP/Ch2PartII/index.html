2.3.1 Unsigned Addition

Consider two nonnegative integers x and y, such that $0 ‚â§ x, y < 2^w$. Each of these values can be represented by a w-bit unsigned number. If we compute their sum, however, we have a possible range $0 ‚â§ x + y ‚â§ 2^{w+1} ‚àí 2$. Representing this sum could require w + 1 bits.

If we were to maintain the sum as a $(w + 1)$-bit number and add it to another value, we may require $w + 2$ bits, and so on. This continued ‚Äúword size inflation‚Äù means we cannot place any bound on the word size required to fully represent the results of arithmetic operations.

Some programming languages, such as Lisp, actually support arbitrary size arithmetic to allow integers of any size .More commonly, programming languages support fixed-size arithmetic.

**Let us define the operation $+^u_w$ for arguments x and y, where 0 ‚â§ x, y < $2^w$, as the result of truncating the integer sum x + y to be w bits long and then viewing the result as an unsigned number. **

This can be characterized as a form of modular arithmetic, computing the sum modulo $2^w$ by simply discarding any bits with weight greater than $2^{w‚àí1}$ in the bit-level representation of x + y.

principle: Unsigned addition

For x and y such that $0 ‚â§ x,y < 2^w$:
$$
x + y = \begin{cases}
x+y , & \text{normal}\
x+y-2^w , &\text{overflow} , 2^w \le x+y < 2 ^{w+1}
\end{cases}
$$

principle: Detecting overflow of unsigned addition

For x and y in the range $0 ‚â§ x,y ‚â§ UMax_w$, let $s = x +^u_w y$. Then the computation of s overflowed if and only if :
$$
s < x
$$
or equivalently:
$$
s < y
$$
derivation:

If the computation is normal
$$
s = x+y \ge x
$$
Also, for w bits data
$$
UMax_w = 2^w-1 < 2^w
$$
If overflow happens, s must smaller than x(y).

2.3.2 Two‚Äôs-Complement Addition

Given integer values x and y in the range $‚àí2^{w‚àí1} ‚â§ x, y ‚â§ 2^{w‚àí1} ‚àí 1$, their sum is in the range $‚àí2^w ‚â§ x + y ‚â§ 2^{w} ‚àí 2$, potentially requiring w + 1 bits to represent exactly.

We truncate the result by dropping the leading bit, then view it as 2‚Äôs complement

principle: Two‚Äôs-complement addition

For integer values x and y in the range $‚àí2^{w‚àí1} ‚â§ x,y ‚â§ 2^{w‚àí1} ‚àí 1$:
$$
x +^t_w y = \begin{cases}
x+y-2^w , & 2^{w-1} \le x+y , &\text{positive overflow}\
x+y , &‚àí2^{w‚àí1} ‚â§ x+y ‚â§ 2^{w‚àí1} ‚àí 1 , &\text{normal}\
x+y+2^w , &x+y < -2^{w-1} , &\text{negative overflow}
\end{cases}
$$
The w-bit two‚Äôs-complement sum of two numbers has the exact SAME bit-level representation as the unsigned sum.

Derivation:

Since two‚Äôs-complement addition has the exact same bit-level representation as unsigned addition, we can characterize the operation $+^t_w$ as one of converting its arguments to unsigned, performing unsigned addition, and then converting back to two‚Äôs complement:
$$
x +^t_wy= U2T_w(T2U_w(x)+^u_wT2U_w(y))\
=U2T_w([(x_{w‚àí1}2^w + x + y_{w‚àí1}2^w + y) \mod 2^w])\
=U2T_w(x+y \mod 2^w)
$$

principle: Detecting overflow in two‚Äôs-complement addition

For x and y in the range $TMin_w ‚â§ x,y ‚â§ TMax_w$, let $s = x +^t_w y$. Then the computation of s

  • has had positive overflow if and only if x > 0 and y > 0 but s ‚â§ 0.
  • has had negative overflow if and only if x < 0 and y < 0 but s ‚â• 0.

Instead, unsigned multiplication in C is defined to yield the w-bit value given by the low-order w bits of the 2w-bit integer product.

We just truncate the high-order w bits.

2.3.6 Multiplying by Constants

Historically, the integer multiply instruction on many machines was fairly slow, requiring 10 or more clock cycles, whereas other integer operations‚Äîsuch as addition, subtraction, bit-level operations, and shifting‚Äîrequired only 1 clock cycle.

As a consequence, one important optimization used by compilers is to attempt to replace multiplications by constant factors with combinations of shift and addition operations.

principle: Multiplication by a power of 2

Let x be the unsigned integer represented by bit pattern $[x_{w‚àí1}, x_{w‚àí2},‚Ä¶,x_0]$. Then for any k ‚â• 0, the w + k-bit unsigned representation of $x2^k$ is given by $[x_{w‚àí1}, x_{w‚àí2},‚Ä¶,x_0, 0,‚Ä¶, 0]$, where k zeros have been added to the right.

principle: Unsigned multiplication by a power of 2

For C variables x and k with unsigned values x and k, such that 0 ‚â§ k < w, the C expression x << k yields the value $x *^u_w 2^k$

Since the bit-level operation of fixed-size two‚Äôs-complement arithmetic is equivalent to that for unsigned arithmetic, we can make a similar statement about the relationship between left shifts and multiplication by a power of 2 for two‚Äôscomplement arithmetic:

principle: Two‚Äôs-complement multiplication by a power of 2

For C variables x and k with two‚Äôs-complement value x and unsigned value k, such that 0 ‚â§ k < w , the C expression x << k yields the value $x *^t_w 2 ^ k $.

For example, suppose a program contains the expression x*14. Recognizing that $14 = 2^3 + 2^2 + 2^1$, the compiler can rewrite the multiplication as (x<<3) + (x<<2) + (x<<1), replacing one multiplication with three shifts and two additions

Even better, the compiler can also use the property $14 = 2^4 ‚àí 2^1$ to rewrite the multiplication as (x<<4) - (x<<1), requiring only two shifts and a subtraction.
